# FHS 目录树安装/卸载脚本

## 功能概述

本脚本用于将项目目录下的文件/文件夹软链接到用户级别 FHS 目录（如 `~/.local/bin/`、`~/.config/`、`~/.local/share/` 等），支持安装和卸载操作。

## 配置说明

脚本通过 `FILES_CONFIG` 列表配置需要管理的文件/文件夹：

```python
FILES_CONFIG: List[Dict[str, str]] = [
    {"path": ".local/bin/nvim", "link_name": "nvim"},
    {"path": ".local/share/nvim", "link_name": "share"},
    {"path": ".config/nvim", "link_name": "config"},
]
```

### 配置项属性

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `path` | string | 是 | 源文件/文件夹的相对路径，同时也是目标路径（相对于用户目录 `~`） |
| `link_name` | string | 否 | 如果不为空字符串，会在脚本目录下额外创建一个名为 `link_name` 的软链接指向源文件 |

### 配置注意事项

- `link_name` 不能与源文件路径指向同一位置（脚本会自动检测并阻止）
- 路径必须遵循 FHS 规范
- 支持文件和文件夹

## 命令行参数

| 参数 | 简写 | 功能 |
|------|------|------|
| `--install` | `-i` | 执行安装（默认行为） |
| `--force` | `-f` | 强制安装，覆盖已存在的文件/链接 |
| `--uninstall` | `-u` | 执行卸载 |
| `--backup` | `-b` | 创建备份（可单独使用） |
| `--help` | `-h` | 显示帮助信息 |

## 安装流程

1. **备份（可选）**：如果指定了 `-b` 参数，先创建文件夹备份
2. **安全检查**：检测 `link_name` 是否会覆盖源文件，若会则报错退出
3. **冲突检测**：检查目标位置是否已存在文件/链接（非强制模式下）
4. **创建目录**：自动创建目标路径的父目录
5. **创建链接**：
   - 在用户目录下创建软链接（`~/{path} -> {脚本目录}/{path}`）
   - 如果配置了 `link_name`，在脚本目录下创建额外软链接（`{脚本目录}/{link_name} -> {脚本目录}/{path}`）

## 卸载流程

1. **备份（可选）**：如果指定了 `-b` 参数，先创建文件夹备份
2. 删除用户目录下的对应软链接（`~/{path}`）
3. 如果配置了 `link_name`，删除脚本目录下的额外软链接
4. 不存在的链接会被忽略（类似 `rm -f`）

## 备份功能

使用 `-b` 参数时，脚本会在执行安装/卸载前创建备份：

- **备份文件名格式**：`{文件夹名}-{日期}-{时间}.tar.gz`
  - 例如：`lazyvim-2026-05-27-185230.tar.gz`
- **备份位置**：脚本文件夹的同级目录
- **备份内容**：整个脚本文件夹，但会排除：
  - 所有软链接
  - 通过 `link_name` 创建的额外软链接

## 使用示例

```bash
# 默认安装（无参）
python install.py

# 显式安装
python install.py -i

# 强制安装（覆盖已存在文件）
python install.py -i -f

# 卸载
python install.py -u

# 显示帮助
python install.py -h

# 安装前备份
python install.py -i -b

# 卸载前备份
python install.py -u -b

# 强制安装并备份
python install.py -i -f -b
```

## 幂等性保障

- **安装**：非强制模式下，已存在的文件会阻止安装；强制模式下会先删除旧链接再创建新链接
- **卸载**：只删除软链接，不会删除真实文件；不存在的链接会被忽略
- **安全检查**：安装前检测配置是否存在自引用风险

## 注意事项

1. 确保脚本有足够权限读写目标目录
2. `link_name` 为空字符串时不会创建额外软链接
3. 卸载时只会删除软链接，不会删除源文件
4. 配置变更后重新运行安装即可更新链接
